## 1. 目的

ローカルファイルシステム上の複数のドキュメントファイル（.docx, .md, .html, .txt）を「起・承・転・結」の 4 構成に沿って統合し、1 つの Markdown ファイル（.md）として出力・編集・保存するオフラインデスクトップアプリケーションを提供する。

---

## 2. スコープ

### In Scope

- ローカルファイル（.docx, .md, .html, .txt）の読み込みと Markdown 変換
- 起承転結スロットへの D&D 配置
- フォルダスキャンによる自動配置
- 配置の確認・並び替え・削除・手動追加
- 統合結果の Markdown エディタ表示・直接編集
- ローカルファイルシステムへの .md ファイル保存
- オフライン完結（インターネット接続不要）

### Out of Scope

- リアルタイム共同編集
- バージョン管理 / 差分比較
- クラウドストレージ連携（初期リリース）
- モバイル最適化

## 4. 機能仕様

### 4.1 ファイル入力・構成管理

#### F-01: フォルダスキャン＆自動配置

```
入力: ローカルフォルダパス
処理:
  1. フォルダ内の対応ファイル一覧を取得 (.docx, .md, .html, .txt)
  2. ファイル名に「起」「承」「転」「結」キーワードが含まれるか判定
  3. 該当スロットへ自動配置
  4. 未分類ファイルは未配置リストに表示
出力: 各スロットに配置されたファイル一覧
```

#### F-02: D&D スロット配置

- @dnd-kit を使用
- スロット間の移動、スロット内の並び替えが可能
- 空のスロットへの新規追加をサポート

#### F-03: 構成の確認・調整

- 各スロットの配置ファイル一覧を表示
- 並び替え（スロット内 / スロット間）
- 削除ボタン（ファイルをスロットから除外）
- 追加ボタン（Electron ファイル選択ダイアログ）

### 4.2 テキスト統合・変換

#### F-04: ドキュメント → Markdown 変換

対応する変換ルール:

| ドキュメント要素 | Markdown 記法 |
|:-----------------|:-------------|
| 見出し 1〜6 | `# ` 〜 `###### ` |
| 太字 | `**text**` |
| 斜体 | `*text*` |
| 箇条書き | `- item` |
| 番号付きリスト | `1. item` |
| リンク | `[text](url)` |
| インラインコード | `` `code` `` |
| テーブル | Markdown テーブル記法 |
| 画像 | `![alt](url)` |

#### F-05: シーケンシャル統合

```
結果 = 起のMD + 区切り + 承のMD + 区切り + 転のMD + 区切り + 結のMD
区切り = "\n\n---\n\n"
```

### 4.3 編集・プレビュー

#### F-06: Markdown エディタ

- CodeMirror 6 ベースのエディタ
- Markdown シンタックスハイライト
- 行番号表示
- 基本的な書式ツールバー（太字、斜体、見出し、リスト）

### 4.4 保存・出力

#### F-07: ローカル保存

```
入力: ファイル名、保存先フォルダパス、Markdown テキスト
処理:
  1. 指定フォルダの存在確認
  2. 同名ファイルの存在チェック（上書き確認ダイアログ）
  3. .md ファイルとしてローカルに保存
出力: 保存成功通知 + ファイルパス
```

## 6. データモデル

### SlotConfig

```typescript
interface SlotConfig {
  type: '起' | '承' | '転' | '結';
  files: LocalFile[];
}

interface LocalFile {
  path: string;        // ローカルファイルパス
  name: string;        // ファイル名
  extension: string;   // .docx | .md | .html | .txt
  size: number;        // バイト数
  lastModified: Date;
}

interface EditorState {
  slots: SlotConfig[];
  mergedMarkdown: string;
  outputFileName: string;
  outputFolderPath: string;
}
```

## 7. IPC API 設計 (Electron Main ↔ Renderer)

| チャンネル | 入力 | 出力 | 説明 |
|:--------|:-----|:-----|:-----|
| `file:scanFolder` | フォルダパス | `SlotConfig[]` | フォルダスキャン＆自動分類 |
| `file:convertToMarkdown` | ファイルパス | `string` | 単一ファイルの MD 変換 |
| `file:mergeDocuments` | `SlotConfig[]` | `string` | 複数ファイルの統合 MD |
| `file:save` | MD文字列, ファイル名, フォルダパス | `{path}` | ローカル保存 |
| `dialog:openFolder` | なし | `string \| null` | フォルダ選択ダイアログ |
| `dialog:openFiles` | 拡張子フィルタ | `LocalFile[]` | ファイル選択ダイアログ |
| `dialog:saveFile` | デフォルト名 | `string \| null` | 保存先選択ダイアログ |

## 8. 非機能要件

| 項目 | 要件 |
|:-----|:-----|
| パフォーマンス | 10ファイルの統合変換を3秒以内に完了 |
| セキュリティ | ローカルファイルのみ操作。ネットワーク通信なし |
| OS 対応 | macOS, Windows, Linux（Electron によるクロスプラットフォーム） |
| データ保持 | 編集中データはアプリ内メモリに保持。永続化はローカルファイルシステム |
| エラーハンドリング | ファイル I/O エラー時にダイアログで通知。リトライ可能な操作はリトライボタンを表示 |

## 10. リスク

| リスク | 影響 | 対策 |
|:-------|:-----|:-----|
| Electron アプリサイズ | インストーラーが大きくなる可能性 | 不要な依存の排除 + electron-builder 最適化 |
| .docx の複雑な書式 | mammoth.js で変換精度が下がる場合がある | 対応書式を明示し、段階的に拡張 |
| OS 間の差異 | パス区切り文字、フォント等の差異 | path モジュール使用 + CI でマルチOS テスト |
| 大容量ファイル | メモリ消費が増大 | ストリーム処理 / ファイルサイズ制限（警告） |
