---

## 1. 目的

Google ドライブ上の複数の Google ドキュメントを「起・承・転・結」の 4 構成に沿って統合し、1 つの Markdown ファイル（.md）として出力・編集・保存するツールを提供する。

---

## 2. スコープ

### In Scope

- Google ドキュメントの読み込みと Markdown 変換
- 起承転結スロットへの D&D 配置
- フォルダスキャンによる自動配置
- 配置の確認・並び替え・削除・手動追加
- 統合結果の Markdown エディタ表示・直接編集
- Google ドライブへの .md ファイル保存

### Out of Scope

- リアルタイム共同編集
- バージョン管理 / 差分比較
- Google ドキュメント以外のファイル形式対応
- モバイル最適化（初期リリース）

## 4. 機能仕様

### 4.1 ファイル入力・構成管理

#### F-01: フォルダスキャン＆自動配置

```
入力: Google ドライブ フォルダ ID
処理:
  1. フォルダ内の Google ドキュメント一覧を取得
  2. ファイル名に「起」「承」「転」「結」キーワードが含まれるか判定
  3. 該当スロットへ自動配置
  4. 未分類ファイルは未配置リストに表示
出力: 各スロットに配置されたファイル一覧
```

#### F-02: D&D スロット配置

- Sortable.js を使用
- スロット間の移動、スロット内の並び替えが可能
- 空のスロットへの新規追加をサポート

#### F-03: 構成の確認・調整

- 各スロットの配置ファイル一覧を表示
- 並び替え（スロット内 / スロット間）
- 削除ボタン（ファイルをスロットから除外）
- 追加ボタン（Google ドライブ Picker で選択）

### 4.2 テキスト統合・変換

#### F-04: Google Doc → Markdown 変換

対応する変換ルール:

| Google Docs 要素 | Markdown 記法 |
|:-----------------|:-------------|
| 見出し 1〜6 | `# ` 〜 `###### ` |
| 太字 | `**text**` |
| 斜体 | `*text*` |
| 箇条書き | `- item` |
| 番号付きリスト | `1. item` |
| リンク | `[text](url)` |
| インラインコード | `` `code` `` |
| テーブル | Markdown テーブル記法 |
| 画像 | `![alt](url)` |

#### F-05: シーケンシャル統合

```
結果 = 起のMD + 区切り + 承のMD + 区切り + 転のMD + 区切り + 結のMD
区切り = "\n\n---\n\n"
```

### 4.3 編集・プレビュー

#### F-06: Markdown エディタ

- CodeMirror 6 ベースのエディタ
- Markdown シンタックスハイライト
- 行番号表示
- 基本的な書式ツールバー（太字、斜体、見出し、リスト）

### 4.4 保存・出力

#### F-07: Google ドライブ保存

```
入力: ファイル名、保存先フォルダ ID、Markdown テキスト
処理:
  1. 指定フォルダの存在確認
  2. 同名ファイルの存在チェック（上書き確認）
  3. .md ファイルとして保存
出力: 保存成功通知 + ファイルURL
```

## 6. データモデル

### SlotConfig

```typescript
interface SlotConfig {
  type: '起' | '承' | '転' | '結';
  files: GoogleDocFile[];
}

interface GoogleDocFile {
  id: string;          // Google Drive File ID
  name: string;        // ファイル名
  mimeType: string;    // application/vnd.google-apps.document
  lastModified: Date;
}

interface EditorState {
  slots: SlotConfig[];
  mergedMarkdown: string;
  outputFileName: string;
  outputFolderId: string;
}
```

## 7. API 設計 (GAS Server Functions)

| 関数名 | 入力 | 出力 | 説明 |
|:--------|:-----|:-----|:-----|
| `scanFolder(folderId)` | フォルダID | `SlotConfig[]` | フォルダスキャン＆自動分類 |
| `getDocAsMarkdown(docId)` | ドキュメントID | `string` | 単一ドキュメントの MD 変換 |
| `mergeDocuments(slots)` | `SlotConfig[]` | `string` | 複数ドキュメントの統合 MD |
| `saveTorive(md, name, folderId)` | MD文字列, ファイル名, フォルダID | `{url, id}` | Google ドライブ保存 |
| `listFolders()` | なし | `Folder[]` | フォルダ一覧取得 |
| `pickFile()` | なし | `GoogleDocFile` | ファイル選択（Picker API） |

## 8. 非機能要件

| 項目 | 要件 |
|:-----|:-----|
| パフォーマンス | 10ファイルの統合変換を5秒以内に完了 |
| セキュリティ | Google OAuth 2.0 によるアクセス制御。GAS の組み込み認証を利用 |
| ブラウザ対応 | Chrome 最新版（主要対象）、Edge、Firefox |
| データ保持 | 編集中データはブラウザセッション内に保持。永続化は Google ドライブのみ |
| エラーハンドリング | API エラー時にユーザーへ通知。リトライ可能な操作はリトライボタンを表示 |

## 10. リスク

| リスク | 影響 | 対策 |
|:-------|:-----|:-----|
| GAS の実行時間制限（6分） | 大量ファイル統合時にタイムアウト | バッチ処理 / ファイル数制限 |
| Google Docs の複雑な書式 | 変換精度の低下 | 対応書式を明示し、段階的に拡張 |
| HtmlService のサイズ制限 | 大きなエディタライブラリの読み込み不可 | CDN 経由でライブラリ読み込み |
| Picker API の制約 | フォルダ選択UIの制限 | カスタムフォルダブラウザの実装 |
